Overview

Wargame: Red Dragon contains over 2,000 units, hundreds of variables, and more than 200,000 data points controlling gameplay mechanics. This tool extracts, processes, and visualises this data to provide accurate statistics, comparisons, and analysis not available in the original game. This tool contains a collection interesting elements visualised from the game and reports the real stats as stats in the in game armoury can contain errors. Data view contains the absolute value, min-max score and z-score.

Key Features

The tool provides a complete analytical view of Wargame: Red Dragon’s underlying data, allowing users to inspect mechanics previously hidden by the game interface. It transforms raw game data into an interactive database, simulator, and analysis platform.

Unit Database

The Unit Database provides a complete view of all available units, exposing the underlying values used by the game engine. Unfortunately in the original game the data presented to the user does contains errors for specific variables. In this software this has been corrected. For data analysis and balancing, it important to see how a single unit compares to the group. It is possible to compare the unit to:

  • the whole dataset
  • the units define role
  • the current filtered list
  • a pool of specfic units

The unit database (Fig.) can view specfic units from the database, and have a varietry of filter options.

The original interface hides how veterancy modifies unit statistics. The tool calculates and displays the modified values dynamically based on the selected veterancy level. The correct values are now displayed (Fig.) right of the arrow depending on the veterancy selected.

Statistical Comparison

A unit can be analysed against multiple reference groups:

  • Entire dataset
  • Assigned role
  • Current filtered selection
  • Custom unit groups

The tool calculates:

  • Absolute values
  • Min/max range
  • Z-score deviation from the selected group

Combat Simulator

The combat simulator is able to select units from the database.

The combat simulator (Fig.) uses extracted unit and weapon statistics to simulate engagements under configurable conditions.

Users can define:

  • Units involved
  • Starting conditions
  • Engagement distance
  • Veterancy level
  • Rules of engagement

The results (Fig.) can be exported as a file with the details selected for clairty on what occured and why.

Weapon Database

In Wargame: Red Dragon, it was not possible to view specfic weapons and which units used them.

Important hidden values like exactly how the Rate of Fire was calculated (Fig.) are displayed. Values such as supply cost are completely hidden altoghter. These hidden values significantly affect gameplay decisions, particularly logistics and resource management. Exposing them allows players to analyse weapon performance and understand the underlying mechanics.

Detection mechanics

Wargame: Red dragon features both detection and stealth mechanics. I made a variety of visualisation of the data.

Detection and stealth mechanics are difficult to evaluate because the game does not expose the relationship between optical strength and stealth values. The visualisation (Fig.) allows users to quickly identify detection thresholds and balance differences.
